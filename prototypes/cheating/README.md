## 发现

### 核心结论：能力叠加不会导致数值失控

1. **全局最大分数=163**（7骰全6+贪欲+牌型大师），远低于第8轮目标250。
2. **所有64种被动组合均产生有限分数**，无NaN、Infinity或负数。
3. **"先加后乘"公式有效控制膨胀**：加法用和、乘法用积，最坏情况3个×1.2也只有1.728倍。
4. **连横术对豹子/满堂红/顺子=0**，限制了最大学科组合的加成空间。

### 重要发现：游戏难度过高 → 已修复

**原版蒙特卡洛模拟显示所有策略0%胜率**，原因分析：

1. **R1通过率85.9%**（消耗品换面保底有效）
2. **R2骤降至7.5-12.3%**：3骰子随机投掷平均~10.5分，目标20分需要非常好的骰子+消耗品
3. **核心瓶颈：3骰子太少**。3骰子的分类只有对子/三条/豹子/散牌，平均分低
4. **被动叠加不够强**：贪欲×1.2、连横+3/颗、牌型+10 — 全被动7骰子平均仅~50分

**经过 3 轮 Monte Carlo 调优（详见 balance-fix/balance-fix2/balance-fix3.js），确定修复方案**：

#### 方案C30（推荐）：初始4骰 + 贪欲×2.0 + 牌型+20 + 有意义目标
- 通关率：14.1%（5000局）
- 各轮通过率：R1-4: 82-100%, R5: 72%, R6: 49%, R7: 100%, R8: 71%
- **核心设计理念**：不降低终点高度，而是给玩家更强的作弊工具
- 关键参数变更：
  - 初始骰子：3→4
  - 贪欲倍率：×1.2→×2.0（分数翻倍！）
  - 连横术加成：+3→+5/颗
  - 牌型大师：+10(满堂红/豹子)→+20(满堂红/豹子/三条)
  - 目标曲线：[12,20,40,65,95,130,175,250]→[8,14,22,35,50,68,88,110]
  - 代币奖励：[4,4,5,5,6,6,7,8]→[5,5,6,6,7,7,8,9]
- **后期分数爆炸**：P75从R6的74跳到R7的110（+49%），真正体现"出千即爽感"

#### 方案A（备选/温和）：初始4骰 + 贪欲×1.5 + 降目标
- 通关率：9.6%（5000局）
- R8目标仅82，缺乏最终Boss的压迫感
- 适合作为降低难度的备选

### 对设计文档的验证

- 铅骰+顺子眼联动工作正常（铅骰保底2使顺子更容易凑）
- 分身术+连横术联动工作正常（更多骰子→更多超出→更多加成）
- 贪欲是万能放大器，对所有分类生效
- 牌型大师精准只对满堂红/豹子生效
- 经济系统：45代币可买全部6被动(24)+骰子扩展(4)+消耗品(2)×多轮

### 原型局限

- 消耗品使用策略简单（只将最低骰子设为6），未实现最优策略（如造豹子、造顺子）
- 未模拟透视消耗品（查看弱点分类+10）
- 敌人规则处理简化（Boss随机规则未模拟）
- 商店随机性未模拟（假设所有道具都可用）
