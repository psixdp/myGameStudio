## 发现

### 核心结论：设计可行

1. **分类匹配算法正确**：穷举验证了3-5颗骰子的所有组合（216+1296+7776=9288种），匹配结果与参考实现完全一致。

2. **分数计算正确**：基础分、加法加成、乘法倍率的"先加后乘"公式在所有测试中输出正确结果。

3. **边界情况处理完善**：
   - 满堂红严格要求恰好两组值（6-7颗骰子不放宽）
   - 连横术对豹子/满堂红/顺子类加成=0（符合设计）
   - 散牌不可被封锁，始终兜底
   - 最低点归零只影响计分不影响分类匹配

4. **浮点精度问题**：乘法倍率叠加时存在浮点误差（如 `1.2×1.5=1.7999...`），需在 floor 前加微小偏移量（1e-9）修正。生产代码需注意此问题。

### 对设计文档的验证

- 优先级排序（豹子>满堂红>大顺>小顺>三条>对子>散牌）工作正常
- 满堂红的"严格3+2"定义在6-7颗骰子时产生了有趣的策略效果（三条后期价值更高）
- 顺子眼能力（允许间隔1）实现了放宽匹配，增加了策略深度
