# Sprint 1 — 基础层 + 核心层

> **日期**: 2026-04-04
> **状态**: 计划中

## Sprint Goal

实现游戏数据基础设施（数据配置 + 随机数）和核心骰子计分循环，为后续功能层提供可依赖的基础。

## Capacity

- 开发节奏: Claude Code 协作开发，按系统逐个推进
- Buffer: 每个系统预留调试时间
- 预计系统数: 4 个（S×3 + L×1）

## Tasks

### Must Have（Critical Path）

| ID | 任务 | 源 GDD | 依赖 | 验收标准 |
|----|------|--------|------|---------|
| S1-01 | 实现数据配置系统 | `design/gdd/data-config.md` | 无 | AC-1~AC-7: JSON 解析正确、引用完整、重复ID警告、统一查询接口、空配置降级 |
| S1-02 | 实现随机数系统 | `design/gdd/rng.md` | S1-01 | AC-1~AC-6: 种子可重现、多流独立、范围正确、权重分布、未初始化报错 |
| S1-03 | 实现骰子系统 | `design/gdd/dice.md` | S1-01, S1-02 | AC-1~AC-10: 投掷合法、setDie/rerollDie/setFloor 正确、临时骰子自动移除、满池拒绝、种子可重现 |
| S1-04 | 实现计分系统 | `design/gdd/scoring.md` | S1-01, S1-03 | AC-1~AC-16: 3-7骰子分类穷举正确、基础分手算一致、加法/乘法叠加、取整、负分保护、散牌保底、封锁分类、连横术豹子=0 |

### Should Have

| ID | 任务 | 说明 | 依赖 | 验收标准 |
|----|------|------|------|---------|
| S1-05 | 计分系统穷举测试 | 3骰216种+4骰1296种全覆盖 | S1-04 | 穷举全部组合与预期一致（原型已有1606测试基础） |
| S1-06 | 集成冒烟测试 | 骰子投掷→计分完整流程 | S1-03, S1-04 | 100次随机投掷计分无报错、分数合理 |

## Carryover from Previous Sprint

无（首个 Sprint）。

## 风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 计分分类匹配边界情况多 | 中 | 高 | 原型已有1606穷举测试可复用，按等价类补充 |
| JSON 配置文件格式需对齐方案C30 | 低 | 中 | 数据配置系统支持 unknown 字段，后期调参不改代码 |
| 骰子面数/数量变化导致计分失效 | 低 | 高 | 4骰~7骰全覆盖测试，参考原型测试用例 |

## 外部依赖

- 无。四个系统均为纯逻辑，不依赖 UI、DOM 或外部服务。

## Definition of Done

- [ ] S1-01 ~ S1-04 全部实现并通过各自 GDD 的验收条件
- [ ] S1-05 穷举测试覆盖 3~7 骰子所有组合
- [ ] S1-06 冒烟测试通过（投掷→计分完整流程）
- [ ] 代码在 `src/` 目录下，按系统分文件
- [ ] 所有测试在 `tests/` 目录下可通过 `npm test` 运行
- [ ] 方案C30 参数已写入配置文件（目标曲线、被动数值、代币奖励）
