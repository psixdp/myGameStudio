## 迭代 1：留骰/重掷（解决问题 1 + 2 的核心）

### 概述

每场战斗从 1 次投掷改为 **2 次投掷 + 中间留骰决策**。

```
当前流程：
  投掷全部 → 敌人搞事 → 用消耗品改写 → 自动计分

改为：
  第一次投掷 → 敌人骰子类规则生效
    → 玩家标记"留/掷"（哪些骰子保留，哪些重掷）
    → 第二次投掷（只重掷未保留的骰子）
    → 用消耗品改写
    → 计分
```

### 策略价值

- **留骰 = 选牌**：拿到 `[3,3,5,6]`，决策留两个 3 冲三条？还是全重掷博更好结果？
- **随机性被策略缓冲**：第一次掷得烂不等于输，第二次可以补救
- **被动能力的策略感涌现**：铅骰让留低值骰子也安心；分身术触发在每次投掷时

### 对现有系统的影响

| 系统 | 改动量 | 说明 |
|------|--------|------|
| 战斗系统 | **中** | 12 步 → ~16 步，增加"留骰决策"和"二次投掷" |
| 骰子系统 | **小** | 新增 `hold(indices)` / `rerollUnheld()` 接口 |
| 计分系统 | **无** | 不改 |
| 敌人系统 | **无** | 不改 |
| 出千能力系统 | **无** | 消耗品仍在投掷后使用 |
| UI 系统 | **中** | 骰子增加"留/掷"交互状态 |

### 战斗流程变更（12 步 → 16 步）

| 步骤 | 名称 | 执行系统 | 说明 |
|------|------|---------|------|
| 1 | 展示敌人信息 | 战斗系统 | 不变 |
| 2 | 第一次投掷 | 骰子系统 | 不变 |
| 2a | 分身术触发（第一次） | 骰子系统 | 复制一颗随机骰子 |
| 3 | 敌人规则（骰子类） | 骰子系统 | 不变 |
| **4** | **留骰决策** | **战斗系统 + UI** | **玩家标记每颗骰子"留"或"掷"** |
| **5** | **第二次投掷** | **骰子系统** | **只重掷未保留的骰子** |
| **5a** | **分身术触发（第二次）** | **骰子系统** | **第二次投掷也触发分身术** |
| 6 | 玩家使用消耗品 | 出千能力系统 | 不变 |
| 7 | 被动托底 | 骰子系统 | 不变 |
| 8 | 分类匹配 | 计分系统 | 不变 |
| 9 | 基础分计算 | 计分系统 | 不变 |
| 10 | 敌人规则（计分类） | 计分系统 | 不变 |
| 11 | 加法加成 | 计分系统 | 不变 |
| 12 | 乘法倍率 | 计分系统 | 不变 |
| 13 | 向下取整 | 计分系统 | 不变 |
| 14 | 胜负判定 | 战斗系统 | 不变 |

### 骰子系统新增接口

```javascript
// 标记骰子为"保留"状态
dicePool.hold(indices)              // indices: 要保留的骰子索引数组

// 重掷所有未保留的骰子
dicePool.rerollUnheld()             // 使用 dice 流

// 重置所有骰子的保留状态（新一轮开始时）
dicePool.clearHolds()

// 查询哪些骰子被保留
dicePool.getHeldIndices()           // 返回被保留的骰子索引数组
```

### 分身术与留骰的交互

- 分身术在第一次投掷和第二次投掷时**各触发一次**（如果拥有该被动）
- 第一次分身术产生的临时骰子：**不能被保留**，第二次投掷后自动消失
- 第二次分身术产生的临时骰子：正常参与后续计分
- 临时骰子在 UI 上用特殊样式标识，留骰阶段显示"临时骰子不可保留"

**⚠️ 平衡风险**：双触发让分身术从"每轮 +1 临时骰"变为"先给信息优势（帮助留骰判断），再给 +1 临时骰"，强度和战术价值都显著提升。需要通过 Monte Carlo 验证当前目标分数曲线是否仍然合理。

### 调优参数

| 参数 | 当前值 | 建议范围 | 影响面 |
|------|--------|---------|-------|
| 每场投掷次数 | 1 → 2 | 1-3 | 策略深度 vs 节奏 |
| 留骰阶段时间限制 | 无 | 无 / 30秒 | 紧张感 vs 沉思 |
| 是否允许全留（不重掷任何骰子） | 是 | 是/否 | "保底"策略的可用性 |
| 是否允许全重掷 | 是 | 是/否 | 极端风险策略的可用性 |

### 新敌人规则机会

| 规则名 | 效果 | 适用敌人 |
|--------|------|---------|
| 强制重掷 | 玩家必须重掷至少 N 颗骰子 | 中后期敌人 |
| 禁止保留 | 留骰阶段跳过，全部重掷 | Boss 规则池 |
| 偷看保留 | 敌人在留骰后、二次投掷前再改写 1 颗 | 高难度敌人 |

## 迭代 2：手动选分类 + 分类差异化奖励

### 概述

取消自动匹配最高分类，让玩家**主动选择目标分类**。同时重新平衡分类奖励，拉开高低级分类的差距。

### 依赖关系

依赖迭代 1（留骰/重掷）完成后实施。留骰决策 + 手动选分类共同构成"我先规划要凑什么型 → 然后留骰 → 然后确认分类"的完整策略链。

### 分类奖励重新设计

| 分类 | 当前奖励 | 建议改为 | 理由 |
|------|---------|---------|------|
| 散牌 | 0 | ×1.0 | 保底，不亏 |
| 对子 | 0 | +2 | 起步分类 |
| 三条 | +5 | +8 | 中期核心 |
| 小顺 | +10 | +15 | 需要规划 |
| 满堂红 | +15 | +25 | 需要牺牲灵活性 |
| 大顺 | +20 | +30 | 高难度高回报 |
| 豹子 | ×3 | ×4 | 终极目标 |

### 分类选择流程

```
投掷完成 → 消耗品改写 → 分类选择阶段
    → 系统列出当前骰子池可匹配的所有分类
    → 默认高亮"理论最高分"分类
    → 玩家可切换到其他合法分类，界面实时预览切换后的分数
    → 玩家确认选择
    → 按选择的分类计算分数
```

**UI 要求**：系统必须帮玩家算好每个分类的预期分（"一目了然"支柱），而不是让玩家心算。

### 分类选择边界情况

| 情况 | 处理方式 |
|------|---------|
| 敌人封锁对子后，对子出现在可选列表中？ | **不可选**。被封分类从可选列表中移除 |
| 强夺令（category_override）和手动选择谁优先？ | 强夺令**跳过选择阶段**，直接强制分类 |
| 可选列表为空（所有分类被封）？ | 不会发生——散牌不可被封，始终兜底 |
| 玩家可以选择散牌吗？ | **可以**。散牌作为合法低级分类，为降维打击提供空间 |
| 透视弱点分类在手动选中怎么体现？ | 选中弱点分类时预览分数中标注"+10 弱点加成" |
| 牌型大师、连横术等依赖分类的能力怎么计算？ | 按玩家选择的分类计算，而非系统最优分类 |

### 新被动能力设计

| 新被动 | 效果 | 费用 | 配合流派 |
|--------|------|------|---------|
| 对子之王 | 对子分类奖励翻倍 | 3 | 稳健流 |
| 顺子直觉 | 选择顺子时额外 +10 | 3 | 顺子流 |
| 三条专家 | 选择三条时骰子点数 ×1.5 | 4 | 连横流 |
| 豹子猎手 | 选择豹子时，允许有 1 颗骰子不同 | 6 | 追梦流 |
| **藏拙** | **选择的分类比可匹配的最高分类低几级，每低一级 +8** | 4 | **降维打击流** |

### 保留旧连横术 + 新增"藏拙"

```
连横术（不变）：超出分类最低要求的每颗匹配骰子 +5
藏拙（新增）：选择的分类比可匹配的最高分类低几级，每低一级 +8
```

两个能力可以叠加——连横术奖励超额匹配，藏拙奖励主动降维。这避免了重做已有能力，同时创造了新的策略空间。

## 迭代 3：流派整理 + 软性羁绊

### 概述

将现有 15 个被动按 4-5 个"流派方向"归类，补充少量缺口能力，建立软性羁绊系统。
**不追求扩展数量**，而是让现有能力形成可读、可追求的路线。

### 依赖关系

依赖迭代 2（手动选分类）。流派方向需要分类差异化奖励才有意义。

### 设计原则（采纳 Gemini 反馈）

**不使用硬性前置依赖**。3 格商店 + 硬前置 = 高死档率。

改为**软性羁绊/涌现式设计**：
1. **独立可用**：每个能力都有独立的基础价值，不需要其他能力就能生效
2. **协同增强**：如果检测到玩家持有同家族的其他能力，激活额外效果（1+1>2）
3. **权重引导**：买了某家族能力后，同家族其他能力在商店出现概率 +50%（已有权重机制支持）

### 流派方向整理

```
┌─ 霸道流（追求大点数）
│   铅骰（最低为2）              已有
│   天降骰（+15基础分）          已有
│   · 协同：同时持有铅骰+天降骰 → 基础分额外 +5
│
├─ 连横流（追求超额匹配）
│   连横术（超出+5/颗）          已有
│   藏拙（降维打击 +8/级）       迭代2新增
│   · 协同：同时持有连横术+藏拙 → 藏拙加成 +2/级
│
├─ 顺子流（追求连续序列）
│   顺子眼（间隔允许1）          已有
│   顺子直觉（顺子+10）          迭代2新增
│   · 协同：同时持有时 → 小顺也享受大顺奖励
│
├─ 花式流（追求高级分类）
│   牌型大师（高级分类+20）      已有
│   豹子猎手（豹子允许1颗不同）   迭代2新增
│   完美主义者（完美匹配×1.5）   已有
│   · 协同：牌型大师+完美主义者 → 分类奖励 +10
│
└─ 万能流（全局放大）
    贪欲（×2.0）                 已有
    反转审判（85%目标即赢）       已有
    强夺令（强制分类）            已有
    · 协同：贪欲+反转审判 → 阈值降至 80%
```

### 新消耗品补充（按需）

| 新消耗品 | 效果 | 费用 | 配合流派 |
|----------|------|------|---------|
| 定向 | 指定一颗骰子重掷，结果必须 ≥3 | 2 | 霸道流 |
| 模仿 | 复制一颗骰子的值到另一颗 | 2 | 花式流 |

### 商店调整

- 流派标注：每个能力标注所属流派方向（Tag 系统）
- **不设置前置门槛**：所有道具始终可购买，不灰显
- 权重引导：已购买某流派能力后，同流派其他能力出现概率 +50%
