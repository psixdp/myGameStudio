# 游戏概念：千王骰局

## 概述

千王骰局是一款类 Balatro 的 Roguelike 骰子构筑游戏。玩家在一局游戏中通过8轮战斗，
每轮投掷骰子并使用"出千"能力改写结果来击败敌人。核心体验是"打破规则的快感"——
从3个普通骰子开始，通过出千商店不断获得作弊能力，最终骰子和分数都变得离谱。
一局约15-20分钟，失败即重来，追求"再来一局"的上瘾循环。

## 玩家幻想

"我不是在赌，我是在作弊，而且我作弊的本事越来越离谱。"

玩家感受到从小骗子到千王之王的成长弧线。初始每颗骰子都是公平的六面骰，
但随着出千能力叠加，概率、规则、常识统统被踩在脚下。最终骰子不再是骰子，
而是你精心打造的出千武器。

## 类型与灵感来源

- **类型**：Roguelike 骰子构筑
- **主要灵感**：Balatro（卡牌类 Roguelike 构筑）
- **次要灵感**：Dicey Dungeons（骰子战斗）、Yahtzee（分类计分）
- **核心差异**："出千/作弊"主题贯穿所有系统——不是在规则内优化，而是直接改写规则

## 核心循环

### 30秒循环（每次投掷）

```
看到敌人目标分数 → 投掷全部骰子(1次) → 看原始结果
    → 选择使用出千能力改写骰子(可选，每轮最多使用2个消耗品)
    → 匹配计分类别计算基础分（自动选择最优分类）
    → 应用被动出千能力的加成和倍率
    → 最终分数 vs 目标分数
```

- 投掷是瞬间动作，核心决策在"出千改写"环节
- 出千改写是即时反馈：改一个骰子面，分数实时更新
- 满足感来自"破烂骰子 → 出千操作 → 爆炸分数"的视觉和数值反差

### 3分钟循环（单场战斗）

```
遭遇敌人(显示目标分数和特殊规则)
    → 规划：我有多少出千道具？哪个组合最优？
    → 执行：投掷 + 出千
    → 结算：达标 → 获得代币；未达标 → 游戏结束
    → 商店：花代币购买新的出千能力
```

- "一掷定生死"的紧张感——没有重试，没有保底
- 策略维度：何时使用消耗品（现在用还是留着？）
- 每轮最多使用2个消耗品，防止无限改写，保留决策张力

### 20分钟循环（单局完整体验）

```
第1轮 (低目标) → 商店 → 第2轮 → 商店 → ... → 第8轮 (高目标) → 胜利
                ↘ 游戏结束 (任意轮次) → 新一局
```

- 8轮固定，清晰的目标线
- 每轮难度递增（目标分数逐步提升）
- 商店每次战后开放，成长节奏快

### 跨局循环

- 无元进度（Jam级别）
- 驱动力来自：发现新的出千能力联动、追求更优构筑路径
- "下次我要试试那个联动..."

## 详细规则

### 骰子基础

- 骰子为标准六面骰（点数1-6）
- 初始骰子数：3个
- 最大骰子数：7个（超出上限时无法再添加骰子）
- 每轮投掷次数：1次（双投消耗品可增加）
- 骰子数量范围：3-7个

### 计分系统（分类计分）

**核心规则**：所有分类的基础分统一为"全部骰子点数之和"（包括不参与匹配的骰子），
再加上分类奖励分。这保证即使匹配到低级分类，也不会因为分数过低而无意义。

| 分类 | 匹配条件 | 基础分 | 奖励分 | 最低骰子数 |
|------|---------|--------|--------|-----------|
| **对子** | 存在2+个相同点数的骰子 | 全部骰子点数之和 | 0 | 2 |
| **三条** | 存在3+个相同点数的骰子 | 全部骰子点数之和 | +5 | 3 |
| **小顺** | 存在4个连续值（如1-2-3-4） | 全部骰子点数之和 | +10 | 4 |
| **满堂红** | 存在3个一样 + 另2个一样 | 全部骰子点数之和 | +15 | 5 |
| **大顺** | 存在5个连续值（如2-3-4-5-6） | 全部骰子点数之和 | +20 | 5 |
| **豹子** | 所有骰子点数相同 | 全部骰子点数之和 | ×3 倍率 | 3 |
| **散牌** | 不匹配以上任何分类 | 全部骰子点数之和 | 0 | - |

**示例**：骰子 [6, 6, 3]
- 匹配"对子"：基础分 = 6+6+3 = 15，奖励分 = 0，分类总分 = 15
- 也可匹配"散牌"（但优先级低于对子，不会被选中）

**示例**：骰子 [4, 4, 4, 2]
- 匹配"三条"：基础分 = 4+4+4+2 = 14，奖励分 = +5，分类总分 = 19

**示例**：骰子 [5, 5, 5, 5, 5]（5个骰子）
- 匹配"豹子"：基础分 = 25，奖励分 = ×3 倍率，分类总分 = 75

#### 分类优先级（自动选择规则）

当骰子同时匹配多个分类时，按以下优先级自动选择最高级别：

```
豹子 > 满堂红 > 大顺 > 小顺 > 三条 > 对子 > 散牌
```

**优先级理由**：高稀有度分类排在前面。豹子最难达成所以优先级最高。

#### 初始骰子数对分类可用性的影响

初始只有3个骰子，部分分类在前期不可用：

| 骰子数 | 可用分类 | 不可用分类 |
|--------|---------|-----------|
| 3个（初始） | 对子、三条、豹子、散牌 | 小顺(需4)、满堂红(需5)、大顺(需5) |
| 4个 | + 小顺 | 满堂红(需5)、大顺(需5) |
| 5个 | + 满堂红、大顺 | 无 |
| 6-7个 | 全部可用 | 无 |

**设计意图**：骰子扩展不仅仅是"数字更大"，而是解锁新的分类和构筑路径。
这创造了"先扩展骰子还是先强化现有分类"的购买决策。

#### 对子的特殊处理

3个骰子出现两个相同（如 [3, 3, 5]）→ 匹配对子。
3个骰子全部相同（如 [4, 4, 4]）→ 按优先级匹配三条而非对子。
出现两组对子（如 [3, 3, 5, 5]）→ 按优先级匹配对子（两条对子不等于满堂红）。
满堂红需要严格的3+2结构。

### 计分公式

**完整计分公式**：

```
最终分数 = (分类基础分 + 加法加成总和) × 乘法倍率总和
```

**计算顺序**（先加后乘）：

1. **分类基础分** = 全部骰子点数之和 + 分类奖励分
2. **加法加成** = 所有提供固定数值加成的被动能力之和（如牌型大师 +10）
3. **乘法倍率** = 所有提供倍率的被动能力之积（如 ×1.2 × ×1.5 = ×1.8）

**示例计算**：

骰子 [6, 6, 6]（3个骰子），持有被动能力"牌型大师"和"贪欲"：
1. 匹配豹子，基础分 = 6+6+6 + (×3 倍率) = 18 × 3 = 54
2. 牌型大师（豹子基础分+10）：加法加成 = +10
3. 贪欲（分数+20%）：乘法倍率 = ×1.2
4. 最终分数 = (54 + 10) × 1.2 = 64 × 1.2 = **76.8**（向下取整 = 76）

**向下取整规则**：乘法计算后向下取整为整数。

### 出千商店

#### 货币

- **代币**：击败敌人获得，数量随轮次递增
  - 第1轮：4代币
  - 第2轮：4代币
  - 第3轮：5代币
  - 第4轮：5代币
  - 第5轮：6代币
  - 第6轮：6代币
  - 第7轮：7代币
  - 第8轮：8代币
  - 总计（全通关）：45代币

#### 道具类型

| 类型 | 描述 | 持续时间 |
|------|------|---------|
| **消耗品** | 单次使用，强力即时效果 | 使用一次后消失 |
| **被动能力** | 本局持续生效的增益 | 整局有效 |
| **骰子扩展** | 骰子池+1（不超过上限7） | 整局有效 |

#### 商店规则

- 每次进店展示3个道具
- 刷新费用：1代币（可多次刷新）
- 道具从池中随机抽取
- 骰子扩展在前3轮出现概率提高（引导玩家先解锁分类）

#### 初始资源

- 每局开始时，玩家获得1个免费的"换面"消耗品
- 设计目的：为第一轮提供基本保底，避免纯运气导致的即死体验
- 符合支柱2：消耗品是有限资源，使用它意味着后续战斗少一张底牌

### 出千能力 — 初始设计

#### 消耗品（每轮最多使用2个）

| 名称 | 效果 | 费用 | 联动潜力 |
|------|------|------|---------|
| 换面 | 将一个骰子变为任意指定点数(1-6) | 2 | 万用型：补缺、造豹子、造顺子 |
| 加料 | 重掷一个骰子，保证结果≥4 | 2 | 配合铅骰保底更稳 |
| 透视 | 查看本场的"弱点分类"（该分类+10分） | 1 | 引导出千方向 |
| 双投 | 额外获得一次完整投掷机会 | 3 | 配合分身术骰子更多 |
| 偷梁换柱 | 将最低点数的骰子替换为6 | 3 | 配合连横术效果翻倍 |

#### 被动能力

| 名称 | 效果 | 费用 | 加成类型 | 联动潜力 |
|------|------|------|---------|---------|
| 铅骰 | 所有骰子最低点数为2 | 4 | 修改骰子 | 基础分下限提高，一切联动受益 |
| 分身术 | 每次投掷时临时复制1个随机骰子（仅当次有效，不计入骰子池） | 5 | 修改骰子 | 配合牌型大师/连横术 |
| 连横术 | 超出分类最低要求的每颗匹配骰子+3固定加成 | 4 | 加法加成 | 配合分身术骰子更多，超额更多 |
| 顺子眼 | 顺子允许间隔1（如1-3-4-5算小顺） | 4 | 修改规则 | 配合铅骰更容易出顺 |
| 贪欲 | 最终分数+20%（×1.2） | 3 | 乘法倍率 | 万能放大器，一切联动受益 |
| 牌型大师 | 满堂红和豹子分类+10固定加成 | 4 | 加法加成 | 配合分身术更容易凑满堂红 |

**联动类型说明**：
- **加法加成**：在乘法之前计算，绝对值增长
- **乘法倍率**：在加法之后计算，百分比增长
- 两个同类型加成叠加时：加法用"和"，乘法用"积"

#### 骰子扩展

| 名称 | 效果 | 费用 | 备注 |
|------|------|------|------|
| 备用骰 | 骰子池永久+1 | 4 | 达到7个上限后不再出现 |
| 千王骰 | 骰子池永久+1，新骰子初始固定为6（仅首次） | 6 | 之后每轮正常投掷 |

### 敌人设计

8轮固定结构，每轮一个独特敌人。敌人有目标分数和可能的特殊规则。

| 轮次 | 敌人名称 | 目标分数 | 特殊规则 | 规则详细说明 |
|------|---------|---------|---------|------------|
| 1 | 街头混混 | 12 | 无 | — |
| 2 | 地痞赌徒 | 20 | 无 | — |
| 3 | 地下庄家 | 40 | 封锁对子 | "对子"分类无法匹配，骰子即使出现对子也只能算散牌或三条 |
| 4 | 赌场荷官 | 65 | 最低点归零 | 你点数最低的骰子在计分时视为0点（不影响分类匹配判断，只影响基础分计算） |
| 5 | 老千同行 | 95 | 狸猫换子 | 敌人在你投掷后随机将你1颗骰子重掷一次 |
| 6 | 赌场经理 | 130 | 封印被动 | 购买价格最高的被动能力本轮不生效（多个同价则随机选一个） |
| 7 | 地下赌王 | 175 | 全面压制 | 所有骰子点数-1（最低为1，影响分类匹配和计分） |
| 8 | 千王之王 | 250 | 千王审判 | Boss：从前6个敌人的特殊规则中随机选取2条同时生效 |

**目标分数校准逻辑**：
- 假设玩家每轮平均获得1个新能力
- 基础分随骰子数增长自然提升（3骰子平均10.5 → 5骰子平均17.5）
- 被动加成和倍率使最终分数在后期显著超过基础分
- 前2轮不设特殊规则，让玩家熟悉机制
- 第3轮开始引入规则，逐步增加复杂度

### 敌人特殊规则 — 实施细节

**封锁对子（地下庄家）**：
- 骰子出现对子时，系统跳过"对子"分类，尝试匹配更低优先级的分类
- 如果骰子只满足对子条件，则降级为散牌
- 影响：迫使玩家追求三条或豹子，否则只有散牌保底

**最低点归零（赌场荷官）**：
- 只在计分阶段生效，不影响分类匹配判断
- 如果多个骰子同为最低点数，只将其中1个归零
- 归零的骰子在UI上显示为半透明或红色，明确标识

**狸猫换子（老千同行）**：
- 发生在玩家投掷之后、出千改写之前
- 被重掷的骰子由系统随机选择，玩家可以清楚看到哪颗被换了
- 设计意图：增加出千消耗品的价值——敌人制造问题，你来解决

**封印被动（赌场经理）**：
- "购买价格最高"以实际支付价格为准（贪欲会使费用+1，按实际费用判断）
- 被封印的被动在UI上以灰色/划线显示
- 如果没有被动能力，此规则无效果

**全面压制（地下赌王）**：
- 所有骰子点数-1，最低为1
- 影响分类匹配判断和计分
- 铅骰（最低点为2）在此规则之后生效：骰子先-1再被铅骰托底为2

**千王审判（千王之王 Boss）**：
- 从前6个敌人的规则池中随机选择2条（不重复）
- 每次挑战Boss时规则不同，增加重玩性
- 规则在战斗开始时展示，玩家可以在战斗前看到需要应对什么
- 如果抽到"封锁对子"+"全面压制"，组合效果按顺序依次结算

### 规则结算顺序

每场战斗的完整结算顺序：

```
1. 展示敌人信息（目标分数 + 特殊规则）
2. 玩家投掷全部骰子
3. 敌人特殊规则生效（狸猫换子的重掷、全面压制的-1等）
4. 玩家使用消耗品改写骰子（每轮最多2个）
5. 铅骰托底（最低点数限制生效）
6. 系统自动匹配最高优先级分类
7. 计算分类基础分（含分类奖励分）
8. 敌人规则影响计分（最低点归零等）
9. 应用加法加成（牌型大师、连横术等）
10. 应用乘法倍率（贪欲的×1.2等）
11. 向下取整为最终分数
12. 比较最终分数与目标分数
```

### 消耗品使用规则

- 每轮战斗最多使用2个消耗品
- 消耗品使用顺序由玩家决定
- 前一个消耗品的效果会反映在骰子上，后一个基于修改后的状态
- 透视可以在投掷前使用（查看弱点分类），其他消耗品在投掷后使用
- 双投消耗品：使用后重新投掷全部骰子，之前的消耗品效果清除（骰子回到随机状态）

### 分身术详细规则

- 每次投掷时，系统随机选择骰子池中的1颗骰子，临时复制1份
- 复制的骰子仅当次投掷有效，不计入永久骰子池
- 复制的骰子参与分类匹配和计分
- 如果骰子池已达上限(7)，分身术仍然生效（临时突破上限到8）
- 临时骰子在UI上用特殊样式标识（发光边框等）

### 连横术详细规则

- "超出分类最低要求的每颗匹配骰子+3固定加成"
- 例：5个骰子 [4, 4, 4, 4, 2]，匹配三条（最低要求3个匹配）
  - 超出数量 = 4 - 3 = 1颗额外匹配骰子
  - 加成 = 1 × 3 = +3
- 例：5个骰子 [5, 5, 5, 5, 5]，匹配豹子（最低要求=骰子总数=5）
  - 超出数量 = 0（所有骰子都是匹配的，没有"超出"）
  - 加成 = 0
- 豹子由于要求所有骰子一致，永远不会有"超出"的部分
- 三条是最容易触发连横术的分类（4+骰子时经常超额）

## 边界情况

| 情况 | 处理方式 |
|------|---------|
| 所有骰子被归零（荷官规则+点数全1） | 分类匹配正常执行（骰子仍为1，只是计分为0），匹配豹子时基础分=0，加成仍可叠加 |
| 分身术骰子池已达上限7 | 临时突破到8，投掷结束后恢复为7 |
| 没有消耗品且投掷结果很差 | 散牌保底：即使不匹配任何分类，仍有"全部骰子点数之和"作为最低分数。初始免费换面提供基本保底 |
| 双投消耗品 + 分身术 | 双投后重新投掷全部骰子，分身术对新投掷生效（再复制一个） |
| 贪欲 + 赌场经理封印 | 贪欲使所有物品费用+1，封印判断按实际支付价格（含贪欲加价后） |
| 顺子眼 + 全面压制 | 全面压制先减1（骰子变为0-5），顺子眼再判断间隔，铅骰再托底为2 |
| 对子匹配时出现两组对子 | 取点数和较大的一组作为"匹配"依据（如[5,5,2,2]以5的对子为准） |
| 千王之王抽到互斥规则 | 不存在互斥规则——所有规则组合都是合法的 |
| 连横术 + 豹子 | 豹子要求所有骰子一致，无"超出"部分，连横术对豹子无效（但有牌型大师作为补偿） |
| 牌型大师 + 封锁对子 | 牌型大师只影响满堂红和豹子，不受封锁对子影响 |

## 依赖关系

本概念文档涵盖以下系统，它们之间存在以下依赖：

| 系统 | 依赖 | 说明 |
|------|------|------|
| 计分系统 | ← 骰子系统 | 需要骰子结果作为输入 |
| 计分系统 | ← 出千能力 | 被动能力修改计分公式 |
| 出千能力 | ← 商店系统 | 能力通过商店购买获得 |
| 商店系统 | ← 货币系统 | 代币是商店的输入 |
| 货币系统 | ← 战斗系统 | 代币通过击败敌人获得 |
| 战斗系统 | ← 计分系统 | 分数比较决定胜负 |
| 敌人系统 | ← 计分系统 | 特殊规则影响计分过程 |
| 骰子系统 | ← 出千能力 | 消耗品修改骰子状态 |

**系统流转图**：

```
投掷骰子 → 敌人规则 → 出千改写 → 匹配分类 → 计算分数 → 比较判定 → 获得代币 → 商店购买
    ↑                                                           |
    └───────────── 新一轮战斗 ←──────────────────────────────────┘
```

## 调优参数

以下参数可在平衡测试时调整，已标注建议范围和影响面：

| 参数 | 当前值 | 建议范围 | 影响 |
|------|--------|---------|------|
| 初始骰子数 | 3 | 2-4 | 前期可用分类数、前期难度 |
| 最大骰子数 | 7 | 5-8 | 后期分类复杂度、计分上限 |
| 每轮消耗品上限 | 2 | 1-3 | 出千操控空间、消耗品消耗速度 |
| 第1轮目标分数 | 12 | 8-15 | 前期难度、新手体验 |
| 第8轮目标分数 | 250 | 150-400 | 后期难度上限、联动是否足够爽 |
| 初始免费消耗品 | 1个换面 | 0-2 | 新手友好度 |
| 商店道具数 | 3 | 2-4 | 购买决策复杂度 |
| 刷新费用 | 1代币 | 1-2 | 经济策略深度 |
| 被动叠加方式 | 先加后乘 | 先加后乘/先乘后加 | 数值膨胀速度 |
| 目标分数增长曲线 | 手动定义 | 线性/指数/手动 | 难度曲线手感 |

## 设计支柱

### 支柱1：出千即爽感

每次使用出千能力都应该产生"我靠这也可以？"的反应。
如果一个出千能力不让人感觉"离谱"，它就不够好。

**设计测试**："在X和Y之间犹豫时，选择让玩家感觉更强大的那个。"

### 支柱2：一掷定生死

每次投掷都承载真实的失败风险，没有安全网。
"这一掷可能是我最后的一掷"的紧张感驱动专注和情感投入。
双投消耗品以3代币的代价打破一次规则——昂贵但值得。

**设计测试**："如果一个机制降低了投掷的赌注，它必须付出显著的资源代价。"

### 支柱3：联动 > 单体

最强的效果来自出千能力之间的组合，而非单个能力。
设计每个能力时，至少考虑2种潜在的联动可能。

**设计测试**："评估新能力时，优先考虑联动潜力而非单体强度。"

### 支柱4：一目了然

玩家能在3秒内理解当前场面：目标多少、我有多少、差距多大。

**设计测试**："如果玩家需要心算，说明UI没做好。"

## 反支柱

- **不是概率模拟器** — 出千存在的意义就是让概率变得不重要
- **不是长线养成游戏** — 没有跨局的元进度（Jam级别）
- **不是叙事游戏** — 没有故事、对话或剧情，纯机制驱动
- **不是多人游戏** — 纯单机体验

## 玩家动机分析（自我决定理论）

| 需求 | 满足方式 |
|------|---------|
| **自主性** | 选择购买哪些出千能力、何时使用、何时保留消耗品 |
| **胜任感** | 学习联动机制、优化构筑、击败更高轮次的敌人 |
| **归属感** | （最低限度）与朋友分享分数/策略 |

## MDA分析

| 层级 | 描述 |
|------|------|
| **机制层** | 骰子投掷、分类计分规则、出千能力（消耗品+被动）、商店、可变骰子数 |
| **动态层** | 构筑联动发现、资源管理（消耗品的存与用）、每场战斗的风险评估 |
| **体验层** | **感官刺激**（看分数爆炸）、**挑战**（一击即死的紧张）、**发现**（找到新联动）、**表达**（打造独特构筑） |

## 心流设计

- **挑战曲线**：目标分数增长速度超过自然战力增长，要求聪明的购买决策和联动利用
- **技巧上限**：识别最优能力组合和消耗品使用时机
- **失败恢复**：快速重开（1秒内开始新局），重试无惩罚
- **"再来一局"触发器**："我只需要早点买到X能力，然后试试Y联动..."

| 方面 | 选择 |
|------|------|
| **平台** | 网页浏览器（HTML/CSS/JS） |
| **渲染** | 基于DOM（CSS动画渲染骰子） |

## 范围层级

### MVP（必须有）

- 3轮战斗，5种出千能力，基础计分，基础商店
- 验证目标：核心循环好玩吗？

### 完整Jam版（应该有）

- 8轮战斗，12-15种出千能力，7种计分类别
- 敌人特殊规则、骰子扩展、消耗品/被动分化

### 扩展目标（最好有）

- 音效、屏幕震动、粒子特效
- 单局统计（最高分、击败敌人数）
- 更多出千能力（20+种）

## 最大风险

1. **联动平衡**：能力联动可能让后期目标分数完全失去威胁 → 通过调优参数表的目标分数曲线调节，必要时设定单轮最高分上限
2. **可变骰子+计分复杂度**：骰子越多，边界情况越多 → 限制最大骰子数为7，充分测试计分规则
3. **单次投掷策略深度**：只掷一次可能感觉过于依赖运气 → 出千能力+消耗品提供操控感，初始免费换面避免纯运气即死
4. **分类自动识别**：可变骰子数量下自动检测最佳分类有组合复杂度 → 明确优先级排序，充分测试

## 验收标准

- [ ] 玩家能在20分钟内完成一局8轮的完整游戏
- [ ] 至少3种不同的"构筑流派"从能力组合中自然涌现
- [ ] 玩家在赢或输之后会自发产生"再来一局"的想法
- [ ] 计分计算在任意骰子数（3-7个）下均正确
- [ ] 没有能力组合能破坏游戏（无限分数、死锁等）
- [ ] MVP能在单次游玩中验证核心循环是否有趣
- [ ] 所有敌人特殊规则有明确的UI反馈（玩家能理解发生了什么）
- [ ] 被动能力叠加始终产生确定性的唯一结果（不依赖顺序）
