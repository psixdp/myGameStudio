# 敌人系统

> **Status**: Designed
> **Author**: user + agents
> **Last Updated**: 2026-04-03
> **Implements Pillar**: 支柱2（一掷定生死——敌人制造压力）

## Overview

敌人系统管理每轮战斗的敌人实体。每个敌人有目标分数和零至多条特殊规则。
系统负责在战斗开始时加载当前轮次的敌人，并在结算流程的对应阶段将规则分发给
骰子系统和计分系统执行。

敌人系统是"规则提供者"而非"规则执行者"——它不直接修改骰子或分数，
而是将规则信息传递给对应系统。

## Player Fantasy

"每个新敌人都带来不同的挑战，我不能用同一套策略从头打到尾。"
从简单的街头混混到拥有组合规则的千王之王，敌人逐步升级压力，
迫使玩家适应和调整出千策略。Boss 的随机规则组合让每次挑战都不一样。

## Detailed Design

### Core Rules

#### 敌人实体

每个敌人由数据配置定义，包含以下字段：

| 字段 | 说明 |
|------|------|
| id | 唯一标识 |
| round | 出现轮次（1-8） |
| name | 显示名称 |
| targetScore | 目标分数 |
| rules | 固定规则ID列表（可为空） |
| bossRule | Boss 专用：从规则池随机抽取配置 |

#### 8轮敌人一览

| 轮次 | 敌人 | 目标分数 | 规则 | 设计意图 |
|------|------|---------|------|---------|
| 1 | 街头混混 | 12 | 无 | 教学关，纯感受投掷和出千 |
| 2 | 地痞赌徒 | 20 | 无 | 巩固基础，开始感受压力 |
| 3 | 地下庄家 | 40 | 封锁对子 | 第一个规则，迫使追求三条+ |
| 4 | 赌场荷官 | 65 | 最低点归零 | 减少基础分，提高出千需求 |
| 5 | 老千同行 | 95 | 狸猫换子 | 动态干扰，增加消耗品价值 |
| 6 | 赌场经理 | 130 | 封印被动 | 惩罚最贵被动，打破依赖 |
| 7 | 地下赌王 | 175 | 全面压制 | 全局减值，铅骰成关键对策 |
| 8 | 千王之王 | 250 | 千王审判（随机×2） | 终极挑战，不可预测 |

#### 规则执行职责划分

敌人系统只"提供"规则，不"执行"规则：

| 规则 | 执行系统 | 敌人系统提供的接口 |
|------|---------|------------------|
| 封锁对子 | 计分系统 | `getBlockedCategories()` → 返回被封锁的分类ID列表 |
| 最低点归零 | 计分系统 | `hasRule('zero_lowest')` → 布尔值 |
| 狸猫换子 | 骰子系统 | `getRuleParams('swap_dice')` → {count: 1, phase: 'post_roll'} |
| 封印被动 | 出千能力系统 | `getSealedPassive()` → 返回被封印的被动能力 |
| 全面压制 | 骰子系统 | `getRuleParams('suppress_all')` → {amount: 1, minValue: 1} |

#### Boss 千王审判

千王之王不使用固定规则，而是在战斗开始时从规则池（前 5 种规则）中随机抽取 2 条：

- 使用 enemy 随机流：`enemyRng.pick(rules, 2, noRepeat: true)`
- 抽取结果在战斗开始时展示给玩家
- 抽到相同类型规则两次不发生（不重复）
- 所有规则组合都是合法的，不存在互斥

### States and Transitions

```
[未激活] --轮次开始--> [展示信息] --玩家确认--> [规则生效中] --结算完成--> [已击败/已结束]
```

- **未激活**：尚未到达该轮次
- **展示信息**：显示敌人名称、目标分数、特殊规则说明
- **规则生效中**：各系统按结算顺序执行规则
- **已击败**：玩家分数 ≥ 目标分数
- **已结束**：玩家分数 < 目标分数（游戏结束）

### Interactions with Other Systems

| 系统 | 交互方向 | 说明 |
|------|---------|------|
| 数据配置系统 | 读 | 读取 enemies.json + enemy-rules.json |
| 随机数系统 | 读 | enemy 流（仅 Boss 千王审判使用） |
| 骰子系统 | 写（通过战斗系统编排） | 提供规则参数，战斗系统调用骰子改写 |
| 计分系统 | 写（通过战斗系统编排） | 提供封锁分类、最低点归零等规则 |
| 出千能力系统 | 写（通过战斗系统编排） | 提供封印被动规则 |
| 战斗系统 | 双向 | 战斗系统协调敌人加载和规则分发 |
| UI系统 | 被读 | 读取敌人名称、目标分数、规则描述 |

## Formulas

敌人系统无公式计算。目标分数从配置读取，规则参数从配置读取。

唯一涉及"计算"的是 Boss 规则随机抽取：

```
bossRules = enemyRng.pick(rulePool, count=2, noRepeat=true)
```

其中 rulePool 为 enemy-rules.json 中所有规则（排除 bossRule 类型，如果有）。

## Edge Cases

| 情况 | 处理方式 |
|------|---------|
| Boss 抽到封锁对子 + 全面压制 | 合法组合，按结算顺序依次执行 |
| Boss 抽到狸猫换子 + 封印被动 | 合法组合，换子在步骤3，封印整轮生效 |
| 玩家没有被动能力时遇到封印被动 | 规则无效果，正常战斗 |
| 玩家没有消耗品时遇到狸猫换子 | 被换了就被换了，无法补救 |
| 敌人配置中 rules 为空数组 | 无特殊规则，纯比分数 |
| enemy-rules.json 中新增规则 | Boss 千王审判自动纳入抽取池 |
| 骰子全为 1 时全面压制 | 减 1 后最低为 1，无变化 |

## Dependencies

### 上游依赖

| 系统 | 依赖性质 | 使用内容 |
|------|---------|---------|
| 数据配置系统 | 硬依赖 | enemies.json + enemy-rules.json |
| 随机数系统 | 硬依赖 | enemy 流（Boss 规则抽取） |

### 下游依赖（被依赖）

| 系统 | 依赖性质 | 读取内容 |
|------|---------|---------|
| 战斗系统 | 硬依赖 | 敌人实体、目标分数、规则列表 |
| 计分系统 | 间接依赖 | 通过战斗系统接收封锁分类、归零规则 |
| 骰子系统 | 间接依赖 | 通过战斗系统接收换子、压制规则 |
| 出千能力系统 | 间接依赖 | 通过战斗系统接收封印被动规则 |
| UI系统 | 软依赖 | 敌人展示信息 |

## Acceptance Criteria

| 编号 | 验收条件 | 验证方式 |
|------|---------|---------|
| AC-1 | 按轮次正确加载对应敌人 | 单元测试：验证每轮加载的敌人 ID 和属性 |
| AC-2 | 目标分数与配置一致 | 单元测试：验证 targetScore |
| AC-3 | 普通敌人返回正确的固定规则 | 单元测试：第3轮返回 block_pair |
| AC-4 | 前2轮敌人无特殊规则 | 单元测试：rules 为空数组 |
| AC-5 | Boss 随机规则不重复 | 单元测试：100 次 Boss 生成，2 条规则不同 |
| AC-6 | Boss 随机规则来自规则池 | 单元测试：验证规则 ID 均存在于 enemy-rules.json |
| AC-7 | 同一种子 Boss 规则一致 | 单元测试：固定种子，验证两局 Boss 规则相同 |
| AC-8 | 无规则敌人正常运行 | 单元测试：前2轮战斗流程完整无报错 |
