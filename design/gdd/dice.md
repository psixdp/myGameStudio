# 骰子系统

> **Status**: Designed
> **Author**: user + agents
> **Last Updated**: 2026-04-03
> **Implements Pillar**: 支柱1（出千即爽感）+ 支柱2（一掷定生死）

## Overview

骰子系统管理骰子池的完整生命周期：初始化、投掷、被出千能力改写、被敌人规则影响。
每个骰子是一个独立的值对象，骰子池是当前所有骰子的有序集合。

核心职责：
- 维护骰子池（3-7个骰子）
- 执行投掷（通过随机数系统的 dice 流）
- 接受外部改写（出千能力修改骰子值）
- 支持临时扩展（分身术临时增加骰子）

## Player Fantasy

"我的骰子从一开始的公平骰子，变成了被我完全操控的作弊武器。"
从3颗普通骰子开始，通过购买扩展和出千能力，骰子逐渐变成离谱的出千工具。
投掷时的紧张感（支柱2）和改写后的爽感（支柱1）都通过骰子状态的视觉变化传递。

## Detailed Design

### Core Rules

#### 骰子池

- 初始骰子数：3 个（从 global-config.json 读取）
- 最大骰子数：7 个（超出上限时无法再添加）
- 每个骰子持有当前面值（1-6），初始化时为未投掷状态
- 骰子池是一个有序数组，骰子按添加顺序排列

#### 投掷

- 每轮投掷次数：1 次（双投消耗品可额外增加 1 次）
- 投掷时所有骰子重新随机（使用 dice 随机流）
- 投掷是原子操作：所有骰子同时出新值
- 双投消耗品：清除当前所有改写效果，重新投掷全部骰子

#### 骰子改写操作

骰子系统提供以下改写接口，供出千能力系统和敌人系统调用：

| 操作 | 效果 | 调用方 |
|------|------|--------|
| `setDie(index, value)` | 将指定骰子设为指定值 | 换面消耗品 |
| `rerollDie(index, minValue)` | 重掷指定骰子，结果 ≥ minValue | 加料消耗品 |
| `rerollRandom(count)` | 随机选择 count 颗骰子重掷 | 狸猫换子（敌人） |
| `setFloor(minValue)` | 所有骰子最低值设为 minValue | 铅骰（被动） |
| `decreaseAll(amount, floor)` | 所有骰子减 amount，最低为 floor | 全面压制（敌人） |
| `replaceLowest(value)` | 最低骰子替换为 value | 偷梁换柱（消耗品） |
| `addTempDie()` | 添加 1 颗临时骰子（随机值） | 分身术（被动） |
| `addPermanentDie(initialValue)` | 永久增加 1 颗骰子 | 备用骰/千王骰（商店） |

#### 临时骰子

- 分身术产生的临时骰子标记为 `isTemp: true`
- 临时骰子参与当轮的分类匹配和计分
- 临时骰子不计入骰子池上限（可突破到 8）
- 当轮结算后自动移除
- 临时骰子在 UI 上用特殊样式标识

### States and Transitions

骰子池在一轮战斗中的状态流转：

```
[空闲] --开始新轮--> [投掷中] --投掷完成--> [已投掷]
                          |                      |
                          |                      +--[敌人规则]--> [规则生效后]
                          |                                      |
                          +--[双投]----> [重新投掷]               +--[出千改写]--> [改写完成]
                          |                                      |
                          +-----------...------------------------+--[铅骰托底]--> [最终状态]
                                                                        |
                                                                  [结算（计分系统接管）]
```

每轮结束后回到 [空闲]，临时骰子移除。

### Interactions with Other Systems

| 系统 | 交互方向 | 接口 |
|------|---------|------|
| 随机数系统 | 读 | `RNG.getStream('dice')` 用于投掷和重掷 |
| 数据配置系统 | 读 | `DataConfig.get('global-config.dice')` 获取骰子数、面数等参数 |
| 计分系统 | 被读 | 计分系统读取骰子池当前值进行分类匹配 |
| 出千能力系统 | 写（被动） | 出千能力调用骰子改写操作 |
| 敌人系统 | 写（被动） | 敌人规则调用骰子改写操作 |
| 战斗系统 | 编排 | 战斗系统按结算顺序协调骰子系统的各阶段 |
| UI系统 | 被读 | UI 读取骰子池状态渲染显示 |

## Formulas

### 投掷单颗骰子

```
dieValue = diceRng.nextInt(minValue, maxValue)
```

默认 minValue=1, maxValue=6，从 global-config.json 读取。

### 加料重掷（保证 ≥4）

```
dieValue = diceRng.nextInt(4, maxValue)
```

### 临时骰子生成

```
tempDieValue = cloneRng.nextInt(minValue, maxValue)
```

使用 clone 随机流（不是 dice 流），确保分身术不影响骰子投掷序列。

### 骰子池点数总和

```
totalValue = dicePool.reduce((sum, die) => sum + die.value, 0)
```

供计分系统使用。

## Edge Cases

| 情况 | 处理方式 |
|------|---------|
| 骰子池已满（7颗）时尝试永久添加 | 操作被拒绝，商店不应出售骰子扩展 |
| 骰子池已满时分身术触发 | 临时骰子允许突破上限到 8，当轮结束移除回 7 |
| 全面压制将骰子减到 0 | 最低为 1，不会出现 0 值 |
| 铅骰 + 全面压制同时存在 | 全面压制先减 1，铅骰再托底为 2（规则结算顺序决定） |
| 最低点归零（敌人规则） | 不修改骰子值本身，由计分系统在计分时将最低骰子视为 0 |
| 换面设为超出 1-6 范围的值 | 钳制到 [1, 6] |
| 双投清除改写后再投掷 | 所有之前的消耗品效果清除，骰子回到纯随机状态 |
| 空骰子池 | 不会发生（初始至少 3 颗），但操作时空数组返回空结果 |

## Dependencies

### 上游依赖

| 系统 | 依赖性质 | 使用内容 |
|------|---------|---------|
| 随机数系统 | 硬依赖 | dice 流（投掷）、clone 流（分身术） |
| 数据配置系统 | 硬依赖 | global-config.dice（骰子数、面数、初始参数） |

### 下游依赖（被依赖）

| 系统 | 依赖性质 | 读取内容 |
|------|---------|---------|
| 计分系统 | 硬依赖 | 骰子池当前值（分类匹配、点数求和） |
| 出千能力系统 | 硬依赖 | 骰子改写接口 |
| 敌人系统 | 硬依赖 | 敌人规则改写接口 |
| 战斗系统 | 硬依赖 | 骰子状态流转（按结算顺序协调） |
| UI系统 | 软依赖 | 骰子池渲染数据 |

## Tuning Knobs

| 参数 | 当前值 | 建议范围 | 影响面 | 来源 |
|------|--------|---------|-------|------|
| 初始骰子数 | 3 | 2-4 | 前期难度、可用分类 | global-config.json |
| 最大骰子数 | 7 | 5-8 | 后期复杂度、计分上限 | global-config.json |
| 骰子面数 | 6 | 4-8 | 随机性范围 | global-config.json |
| 每轮投掷次数 | 1 | 1-2 | 基础随机性 vs 操控感 | global-config.json |
| 分身术临时上限突破 | 8 | 7-10 | 分身术最大效果 | 出千能力配置 |

## Acceptance Criteria

| 编号 | 验收条件 | 验证方式 |
|------|---------|---------|
| AC-1 | 初始化骰子池数量正确 | 单元测试：验证初始骰子数为配置值 |
| AC-2 | 投掷产生合法值（1-6） | 单元测试：10000 次投掷，所有值在范围内 |
| AC-3 | setDie 正确修改指定骰子 | 单元测试：设置后验证值和索引 |
| AC-4 | rerollDie 保证最小值 | 单元测试：100 次加料重掷，结果均 ≥4 |
| AC-5 | setFloor 托底生效 | 单元测试：含 1 的池 setFloor(2) 后无 1 |
| AC-6 | decreaseAll 不低于地板值 | 单元测试：全 1 的池 decreaseAll(1,1) 后仍全 1 |
| AC-7 | 临时骰子添加和自动移除 | 单元测试：添加后验证数量+1，结算后验证回到原数量 |
| AC-8 | 满池时永久添加被拒绝 | 单元测试：7 颗骰子时 addPermanentDie 返回 false |
| AC-9 | 满池时临时骰子可突破 | 单元测试：7 颗骰子时 addTempDie 成功到 8 |
| AC-10 | 同一种子投掷序列可重现 | 单元测试：固定种子，验证两局投掷序列完全一致 |
