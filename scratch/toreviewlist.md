# GLM-5 策略路线图评估与优化建议 (Strategy Roadmap Review)

**评估对象**: `design/gdd/strategy-roadmap.md`
**评估日期**: 2026-05-19

## 1. 整体评价
GLM-5 规划的方案非常优秀，完美切中了游戏目前“缺乏玩家代理（Player Agency）”的核心痛点。引入《快艇骰子》（Yahtzee）式的留骰机制将发生质变。

## 2. 针对“迭代1：留骰/重掷”的建议
*   **极度赞同**：这是将游戏从“自动老虎机”变为“策略构筑”的关键。
*   **UI/UX 风险**：必须在 UI 上极度清晰地区分“常规骰子”、“保留（锁定）的骰子”和“分身术产生的临时骰子”。临时骰子不可保留的规则需要直观的视觉反馈（如灰色锁图标），防止玩家产生 Bug 错觉。

## 3. 针对“迭代2：手动选分类”的建议
*   **设计亮点**：“降维打击”（为了连横术的超额加成，故意选择低级分类）是天才设计，创造了非线性的策略空间。
*   **交互优化**：绝不能让玩家每回合自己心算各分类收益（违背“一目了然”支柱）。系统必须默认高亮“理论最高分”的分类，同时允许玩家手动切换到其他合法分类，并在界面实时预览切换后的最终分数。

## 4. 针对“迭代3：流派家族”的重大修正建议 ⚠️
*   **存在陷阱**：**坚决反对在肉鸽商店中引入“硬性前置依赖”**（如：必须先买A才能买B）。在每次只刷新3个道具的商店中，硬性科技树会导致严重的污染卡池和极高的“死档率”。
*   **替代方案（软性羁绊/涌现式设计）**：
    *   **取消购买门槛**：所有道具都可以独立购买，且具有独立的基础价值。
    *   **权重控制**：买了一个流派的基础道具后，暗中提高该流派后续道具在商店的出现概率（权重机制已在经济系统中支持）。
    *   **协同机制**：道具 B 本身有用，但如果检测到玩家持有道具 A，道具 B 会激活额外效果（1+1>2），鼓励玩家自发收集同家族能力，而不是强制限制。

## 5. 下一步开发落地建议 (Action Items)
1.  将迭代 1 和迭代 2 合并为一个大的玩法重构 Sprint，因为留骰和选分类在玩家心理预期上是连续的操作。
2.  重构 `ui.js` 和 `game-flow.js` 的状态机，增加 `ROLL_HOLD_PHASE` 和 `CATEGORY_SELECT_PHASE`。
3.  废弃迭代 3 中的“前置条件”设计，改为设计基于词缀（Tags）的软性联动系统。
